# 贡献指南

感谢您对 Kali-MCP 项目的关注！我们欢迎各种形式的贡献，包括修复 Bug、改进文档以及添加新的渗透测试工具。

> 要进行重构级别的改动前，建议先和项目维护者联系，联系方式: 
> 邮箱: nraintd@qq.com

## 参与流程

### 1. Fork 仓库

点击 GitHub 页面右上角的 **Fork** 按钮，将项目克隆到你自己的账号下.

### 2. 本地开发

将你 Fork 的仓库克隆到本地：
```bash
git clone https://github.com/your-username/kali-mcp.git
cd kali-mcp
```

### 3. 构建与测试

项目使用 [taskfile](https://taskfile.dev/) (类似 makefile) 来管理构建流程；不过无需安装，它已经作为项目的工具依赖，可以直接通过 `go tool task` 来运行。

- **构建 linux/amd64**：

因为是 kali-mcp，所以肯定是只支持 linux 的。

```bash
go tool prepare
go tool task build
```

- **构建所有 linux 架构（开发模式）**：

开发模式会顺便运行 `go tool task prepare`

```bash
go tool task build:dev
```

- **构建所有 linux 架构（发布模式）**：

```bash
go tool task build:release
```

### 4. 提交修改

**注意：在开始开发或准备提交前，请务必拉取上游仓库的最新代码并合并到你的分支中。**

```bash
git remote add upstream https://github.com/nraintd/kali-mcp.git
git fetch upstream
git merge upstream/main
```

**重要：如果你的改动涉及新功能或重大修复，请务必更新项目根目录下的 `VERSION.env` 文件以反映版本变更。**


```bash
git add <你改动的文件列表>
git commit -m "你的提交信息"
```

### 5. 推送并创建 Pull Request

将代码推送到你的远程分支：

```bash
git push origin your-branch-name
```

然后在 GitHub 上点击 **New pull request**。

*注：普通贡献者无需创建 Git Tag，版本发布和 Tag 的打标将由项目维护者在合入代码后统一处理。*

## 核心开发指南

如果你想为 Kali-MCP 添加新的 Kali 工具支持，请阅读：
👉 [**添加工具指南**](添加工具.md)

### 版本发布流程

项目版本由 `VERSION.env` 文件管理。打版本流程如下：

1. 更新 `VERSION.env` 中的版本号。
2. 提交更改并推送。
3. 创建对应的 Git Tag（如 `v0.4.0`）并推送到 GitHub。
4. GitHub Actions 会自动触发集群构建并发布 Release。

# 添加工具指南

Kali-MCP 的设计目标是易于扩展。每个工具都应该是一个独立的模块，并遵循项目定义的标准接口和开发模式。

## 0. 核心接口 (Tool Interface)

在开发新工具之前，你需要了解 `internal/toolkit/tool.go` 中定义的 `Tool` 接口。每个工具都必须完整实现以下三个方法：

```go
type Tool interface {
	// Name 返回工具在 MCP 协议中的名称
	Name() string
	// Options 定义工具的描述信息以及它所接受的参数列表
	Options() []mcp.ToolOption
	// Handler 是工具的核心逻辑，负责处理请求并执行相应的工具逻辑
	Handler(ctx context.Context, req mcp.CallToolRequest) (*mcp.CallToolResult, error)
}
```

本指南将以 `hydra` 工具的实现为例，带你一步步了解如何按照此接口添加一个新工具。

## 1. 定义结构体 (type)

首先，你需要定义工具的结构体。结构体通常需要包含 `logger` 用于记录日志，以及 `eh` (`*toolkit.ExecHelper`) 用于辅助执行具体的命令。

```go
type Hydra struct {
	logger *slog.Logger
	eh     *toolkit.ExecHelper
}
```

## 2. 构造函数 (New)

构造函数用于初始化工具及其依赖。`ExecHelper` 会统一管理命令执行的超时、输出转换和基础日志。

```go
func NewHydra(exec *executor.Executor, logger *slog.Logger) *Hydra {
	return &Hydra{
		logger: logger, 
		eh: toolkit.NewExecHelper(exec, logger),
	}
}
```

## 3. 工具名称 (Name)

`Name()` 方法返回该工具在 MCP 协议中的名称。

```go
func (h *Hydra) Name() string {
	return "hydra_attack"
}
```

## 4. 参数定义 (Options)

`Options()` 方法使用 `mcp.ToolOption` 定义工具的描述信息和接受的参数。

```go
func (h *Hydra) Options() []mcp.ToolOption {
	return []mcp.ToolOption{
		mcp.WithDescription("Run Hydra password attack against a target service."),
		mcp.WithString("target", mcp.Required()),
		mcp.WithString("service", mcp.Required()),
		mcp.WithNumber("threads", mcp.DefaultNumber(4), mcp.Min(1)),
		mcp.WithString("username", mcp.DefaultString("")),
		mcp.WithString("username_file", mcp.DefaultString("")),
		mcp.WithString("password", mcp.DefaultString("")),
		mcp.WithString("password_file", mcp.DefaultString("")),
		mcp.WithString("additional_args", mcp.DefaultString("")),
	}
}
```

## 5. 逻辑处理 (Handler)

`Handler` 方法是工具的核心实现，必须遵循 `Tool` 接口定义的函数签名：

```go
func (h *Hydra) Handler(ctx context.Context, req mcp.CallToolRequest) (*mcp.CallToolResult, error) {
    // 逻辑处理可以拆解为以下几个部分：
    // ...
}
```

### A. 参数解析与校验 (Params)

利用 `param` 包提供的链式操作进行各种格式化（如 `TrimSpace`）和校验（如 `NotEmpty`, `Gt`, `MutuallyExclusive`）。

**重要提示**：所有的链式校验（如 `NotEmpty`）或跨字段校验（如 `MutuallyExclusive`）并不会立即返回错误，而是将错误缓存在 `ps` 对象内部。**你必须在解析完所有参数后，统一调用 `ps.Err()` 检查错误**。

```go
	ps := param.NewParams(req)

	username := ps.String("username", "").TrimSpace().Parse()
	usernameFile := ps.String("username_file", "").TrimSpace().Parse()
	password := ps.String("password", "").TrimSpace().Parse()
	passwordFile := ps.String("password_file", "").TrimSpace().Parse()

	target := ps.String("target", "").TrimSpace().NotEmpty().Parse()
	service := ps.String("service", "").TrimSpace().NotEmpty().Parse()
	threads := ps.Int("threads", 4).Gt(0).ParseToString()
	additionalArgs := ps.String("additional_args", "").TrimSpace().Parse()

	ps. // username 与 username_file、password 与 password_file 互斥
		MutuallyExclusive("username", "username_file").
		MutuallyExclusive("password", "password_file")

	// 必须执行此检查！如果上述任何解析或校验失败，ps.Err() 将返回第一个遇到的错误
	if err := ps.Err(); err != nil {
		h.logger.Error("invalid parameters for hydra attack", "error", err)
		return mcp.NewToolResultError(err.Error()), nil
	}
```

### B. 命令构建 (CmdBuilder)

使用 `cmdbuilder` 提供的方法处理复杂的条件逻辑，例如 `AddIfElse` 处理参数互斥，以及 `AddSplit` 处理用户传入的自定义 `additional_args`。

```go
	bin, args, err := cmdbuilder.
		NewCmdBuilder("hydra").
		Add("-t", threads).
		AddIfElse(username != "",
			[]string{"-l", username}, []string{"-L", usernameFile}).
		AddIfElse(password != "",
			[]string{"-p", password}, []string{"-P", passwordFile}).
		AddSplit(additionalArgs).Add(target, service).
		Build()
	if err != nil {
		h.logger.Error("failed to build hydra command", "error", err)
		return mcp.NewToolResultError(err.Error()), nil
	}
```

### C. 执行与返回

最后调用 `eh.RunCmd`。它会自动将命令的 stdout/stderr 包装成 MCP 返回格式。

```go
	return h.eh.RunCmd(ctx, bin, args)
```

## 6. 错误处理与日志原则

在 `Handler` 实现中，我们需要遵循统一的错误处理与日志记录规范：

- **错误返回**：对于业务逻辑错误（参数校验失败、命令构建失败），应使用 `mcp.NewToolResultError` 返回给用户，而不是直接返回 `error` 接口，这样客户端才能看到具体的错误信息。
- **日志记录 (Logging)**：
  - **记录什么**：记录那些由用户输入引起的问题（如非法参数组合）或系统配置/构建问题（如 `CmdBuilder` 报错）。
  - **怎么记录**：统一使用 `h.logger.Error`，并附带上下文，以便在后台排查问题。
  - **不要记录什么**：不需要在 `Handler` 中记录命令执行的 stdout/stderr 或成功的调用记录。这些信息由 `ExecHelper` 或底层的 `executor` 在需要时（如 Debug 模式下）统一透明处理。

## 6. 注册工具

最后，在 `internal/tools/all_tools.go` 的 `All` 函数中注册你的新工具：

```go
func All(exec *executor.Executor, logger *slog.Logger) []toolkit.Tool {
	return []toolkit.Tool{
		// ...
		NewHydra(exec, logger),
		// ...
	}
}
```
